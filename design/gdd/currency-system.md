# 재화 시스템 (Currency System)

> **Status**: In Design
> **Author**: user + game-designer
> **Last Updated**: 2026-03-25
> **Implements Pillar**: 체감있는 성장 (Feel the Growth)

## Overview

재화 시스템은 시드머니의 모든 화폐 흐름을 정의하고 관리하는 Foundation 시스템이다.
두 개의 계좌를 운영한다: 플레이어의 영구 자산인 **예수금**과 모의투자 대회에
사용하는 **모의투자 시드**. 예수금은 보육원 퇴소 청년의 정착지원금 100만원에서
시작하여 시즌 상금으로 성장하며, 향후 현물 시장 시뮬레이션의 투자 자금이 된다.
모의투자 시드는 대회 참가 시 별도 지급되는 가상 자금으로, 시즌 종료 시 리셋된다.

## Player Fantasy

플레이어의 예수금 100만원은 보육원 퇴소 청년의 전 재산이다. 이 돈 자체를 걸 수는
없지만, 모의투자 대회에서 실력을 증명하면 상금이 예수금에 쌓인다. "100만원에서
시작해서 예수금 1억을 만들었다" — 이것이 장기 성장의 지표다. 모의투자는 연습이
아니라 진짜 대회이고, 상금은 진짜 내 돈이 된다. 예수금이 충분히 쌓이면 현물 시장에
직접 뛰어들 수 있다 — 그것이 최종 목표다.

## Detailed Design

### Core Rules

1. **계좌 구조**: 두 개의 분리된 계좌가 존재한다.
   - **예수금 (Deposit)**: 플레이어의 영구 자산. 단위: 원 (₩). 시작: 1,000,000원.
   - **모의투자 시드 (Sim Seed)**: 모의투자 대회 전용 가상 자금. 단위: 원 (₩).
     대회 참가 시 1,000,000원 지급. 시즌 종료 시 리셋.

2. **예수금 규칙**:
   - 게임 시작 시 1,000,000원 (정착지원금).
   - 시즌 상금이 여기로 입금된다.
   - 시즌 리셋에도 영구 유지.
   - 소비처 (향후 확장): 현물 시장 시뮬레이션 투자 자금.
   - MVP에서는 증가만 가능 (상금 입금), 소비 시스템 없음.

3. **모의투자 시드 규칙**:
   - 시즌 시작 시 1,000,000원 지급 (예수금과 별개).
   - 시즌 종료 시 리셋 (잔액/수익률 무관하게 초기화).
   - 매수 시 차감, 매도 시 증가.
   - 음수 불가 — 잔액 부족 시 주문 거부.
   - 보유 주식 평가액 + 현금 잔액 = 대회 내 총 자산.

4. **시즌 상금 규칙**:
   - 시즌 종료 시 순위에 따라 상금 지급.
   - 상금은 **원화**로 예수금에 직접 입금.
   - 상금은 모의투자 시드가 아닌 예수금에 합산.

5. **총 자산 (대회 내)** — 포트폴리오 관리 시스템이 계산:
   - `sim_total_assets = sim_cash + sum(stock_quantity * current_price)`
   - 시즌 순위는 이 값 기준.
   - 재화 시스템은 `sim_cash`만 관리하고, 총 자산 계산은 포트폴리오에 위임.

6. **예수금 총액**:
   - `deposit_total = initial_deposit + sum(all_season_prizes)`
   - 장기 성장 지표.

### States and Transitions

| State | Description | Transition |
|-------|-------------|-----------|
| **IDLE** | 시즌 사이. 예수금만 존재. 모의투자 시드 없음 | → SEASON_ACTIVE (시즌 참가 시) |
| **SEASON_ACTIVE** | 시즌 진행 중. 모의투자 시드로 매매 가능 | → SEASON_SETTLED (시즌 종료 시) |
| **SEASON_SETTLED** | 시즌 종료. 대회 총 자산 확정. 상금 계산 | → IDLE (상금 수령 후) |

### Interactions with Other Systems

| System | Direction | Interface |
|--------|-----------|-----------|
| **주문 처리 엔진** | 주문이 이 시스템에 의존 | `get_sim_cash()` → 모의투자 잔액 확인. `sim_deduct(amount)` / `sim_add(amount)` → 매수/매도 시 잔액 변동 |
| **포트폴리오 관리** | 포트폴리오가 이 시스템에 의존 | `get_sim_cash()` → 현금 잔액 제공. 대회 내 총 자산 계산(`sim_cash + 보유 주식 평가액`)은 포트폴리오 시스템이 수행 |
| **시즌/대회 관리** | 시즌이 이 시스템에 의존 | `init_season_seed(amount)` → 시즌 시작 시 시드 지급. `settle_season()` → 시드 리셋. `award_prize(amount)` → 상금을 예수금에 입금 |
| **트레이딩 스크린 (UI)** | UI가 참조 | `get_sim_cash()`, `get_deposit()` → 잔액 표시 |

## Formulas

### 대회 내 총 자산 및 수익률 (참고 — 포트폴리오 관리 시스템 소유)

아래 공식은 포트폴리오 관리 시스템이 수행하는 계산이다. 재화 시스템은
`sim_cash`만 제공하며, 총 자산 집계와 수익률 계산은 포트폴리오가 담당한다.
상세 공식은 포트폴리오 관리 GDD에서 정의한다.

```
sim_total_assets = sim_cash + sum(holdings[i].quantity * holdings[i].current_price)
return_rate = (sim_total_assets - sim_seed) / sim_seed * 100
```

### 시즌 상금 테이블

| Season Rank | Prize (₩) | Rationale |
|-------------|-----------|-----------|
| 1등 | 500,000 | 챔피언 보상. 예수금의 50% 증가 |
| 2등 | 300,000 | — |
| 3등 | 200,000 | — |
| 4-5등 | 100,000 | 상위권 |
| 6-10등 | 50,000 | 참가 보상 |
| 시즌 완주 보너스 | +30,000 | 중도 포기 방지 |

→ 1등 수상 시 예수금: 1,000,000 + 500,000 + 30,000 = 1,530,000원

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| 모의투자 잔액 부족으로 매수 불가 | 주문 거부 + "잔액 부족" 메시지 | 빚 매수 불가 |
| 전 시드를 한 종목에 올인 | 허용. 리스크는 플레이어 판단 | 필라 "판단이 곧 실력" |
| 대회 내 총 자산이 0원 | 시즌 계속 진행 가능. 관전/학습 | 빈손이어도 끝까지 |
| 수익률 1000% 이상 | 허용. 상한선 없음 | 극단적 성공도 재미 |
| 시즌 미참가 | 예수금 변동 없음. 상금 없음 | 참가는 선택 |
| 예수금으로 모의투자 매매 시도 | 불가. 모의투자는 별도 시드만 사용 | 계좌 분리 원칙 |
| 향후 현물 시장에서 예수금 전부 손실 | 예수금 0원 가능. 모의투자 참가는 여전히 가능 (시드 별도 지급) | 모의투자는 항상 접근 가능 |

## Dependencies

| System | Direction | Nature of Dependency |
|--------|-----------|---------------------|
| 주문 처리 엔진 | 주문이 이 시스템에 의존 | 모의투자 잔액 확인/차감/증가. **Hard** |
| 포트폴리오 관리 | 포트폴리오가 이 시스템에 의존 | `get_sim_cash()`로 현금 잔액 조회. 총 자산 계산은 포트폴리오가 수행. **Hard** |
| 시즌/대회 관리 | 시즌이 이 시스템에 의존 | 시드 지급/리셋/상금 입금. **Hard** |
| 트레이딩 스크린 (UI) | UI가 이 시스템에 의존 | 잔액 표시. **Soft** |

이 시스템은 다른 시스템에 의존하지 않는 Foundation 시스템이다.

## Tuning Knobs

| Parameter | Current Value | Safe Range | Effect of Increase | Effect of Decrease |
|-----------|--------------|------------|-------------------|-------------------|
| `initial_deposit` | 1,000,000 | 100,000-10,000,000 | 예수금 시작이 여유로움 | 더 절박한 시작 |
| `sim_seed_amount` | 1,000,000 | 100,000-10,000,000 | 분산 투자 여유 | 집중/올인 강제 |
| `prize_1st` | 500,000 | 100,000-2,000,000 | 예수금 빠르게 성장 | 느린 성장, 장기 플레이 |
| `prize_completion` | 30,000 | 0-100,000 | 완주 인센티브 강화 | — |

## Acceptance Criteria

- [ ] 게임 시작 시 예수금이 정확히 1,000,000원으로 초기화됨
- [ ] 시즌 시작 시 모의투자 시드가 정확히 1,000,000원으로 지급됨
- [ ] 예수금과 모의투자 시드가 완전히 분리되어 상호 간섭 없음
- [ ] 매수 시 모의투자 잔액이 정확히 차감, 매도 시 정확히 증가
- [ ] 모의투자 잔액이 음수가 되는 경우가 절대 없음
- [ ] 시즌 종료 시 순위에 맞는 상금이 예수금에 정확히 입금됨
- [ ] 예수금은 시즌 리셋에도 영구 유지됨
- [ ] 포트폴리오 시스템이 계산하는 대회 내 총 자산에 `get_sim_cash()` 값이 정확히 반영됨
- [ ] 성능: 잔액 조회/변경이 1ms 이내
- [ ] 모든 재화 값은 정수 (원 단위, 소수점 없음)
